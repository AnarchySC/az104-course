Lab 01 - Manage Microsoft Entra ID Identities

Scenario: Organizations need to provision user accounts and groups in Microsoft Entra ID to support authentication for engineering staff.

1

Create and Configure User Accounts

▼

Provision internal and external user accounts in Microsoft Entra ID with appropriate properties and roles.

  1. Access Microsoft Entra ID in the Azure portal.
  2. Navigate to Users > Create new user.
  3. Configure the following settings:
    SettingValue
    User principal nameaz104-user1
    Display nameaz104-user1
    PasswordAuto-generated
    Account enabledYes
  4. Configure additional properties:
    PropertyValue
    Job titleIT Lab Administrator
    DepartmentIT
    Usage locationUnited States
  5. Invite an external user with the following details:
    PropertyValue
    EmailExternal email address
    Display nameExternal user name
    Job titleAs appropriate
    DepartmentAs appropriate
    Usage locationUnited States
2

Create Groups and Add Members

▼

Create a security group, assign ownership, and add members to manage group-based access.

  1. Access Groups in Microsoft Entra ID.
  2. Create a new security group:
    SettingValue
    Group typeSecurity
    Group nameIT Lab Administrators
    Membership typeAssigned
  3. Designate yourself as group owner.
  4. Add az104-user1 and the invited external user as members.
  5. Verify the group creation and membership in the Groups blade.

Key Takeaways

  • Tenants represent organizational instances for managing cloud service access.
  • User account types include internal and guest accounts with role-specific access.
  • Groups support both static (assigned) and dynamic membership models.
  • Security and Microsoft 365 are the primary group classifications.

Demo - Configure Entra ID Identities

Walk through the core Entra ID blades to explore tenant information, user management, and group configuration.

1

Review License and Domain Information

▼
  1. Navigate to the Microsoft Entra ID blade in the Azure portal.
  2. Examine the Overview page to review tenant information, including licensing details and the primary domain.
2

Explore User Accounts

▼
  1. Access the Users blade in Microsoft Entra ID.
  2. Compare the options for "New user" vs "New guest user" to understand the differences.
  3. Create a new user and review the available sections: Identity, Groups, Roles, Settings, and Job info.
  4. Review additional management features: password reset, user deletion, and sign-in logs.
3

Explore Group Accounts

▼
  1. Navigate to the Groups blade in Microsoft Entra ID.
  2. Create a new group or review an existing group.
  3. Examine the membership type options (Assigned vs Dynamic) and group type classifications (Security vs Microsoft 365).

Exercises

Reinforce your understanding of Entra ID identities with these hands-on exercises.

Easy

Exercise 1: Identity Audit

You've been asked to audit your organization's Entra ID tenant.

  1. Navigate to Entra ID and document the primary domain name.
  2. Check how many licensed users exist.
  3. List all user accounts and identify which are internal vs guest.
  4. Document which users have admin roles assigned.
Hint (click to reveal)
The Overview blade shows tenant info. User properties show account type.
Medium

Exercise 2: User Lifecycle Management

Create a complete user onboarding and offboarding workflow.

  1. Create a new user "test-engineer1" with job title "QA Engineer", department "Engineering".
  2. Create a security group "QA Team" and add the user.
  3. Verify the user can be found in the directory.
  4. Disable the user account (simulating offboarding).
  5. Delete the user and verify they appear in Deleted users.
Hint (click to reveal)
Deleted users can be restored within 30 days from the Deleted users blade.
Hard

Exercise 3: Group-Based Access Design

Design a group structure for a company with Engineering, Sales, and Support departments.

  1. Create three security groups, one per department.
  2. Create two users per department with appropriate job titles.
  3. Add users to their respective groups.
  4. Create a cross-functional group "All Staff" containing all users.
  5. Verify membership counts for each group.
  6. Consider: What would change if you used dynamic groups instead of assigned?
Hint (click to reveal)
Dynamic groups require Entra ID Premium P1. They auto-populate based on user attributes like department.
